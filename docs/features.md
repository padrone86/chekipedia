# 機能要件

## Phase 1（最初に作る）

### 1. レギュレーション閲覧

- グループ名で検索し、特典会のレギュレーションを閲覧できる
- 認証不要（誰でも閲覧可能）

### 2. 管理者によるデータ入力

- 管理者（開発者本人）がレギュレーションを登録・編集・削除できる
- 管理者認証が必要

## Phase 2

### 3. Twitter（X）認証・ユーザー登録

- Amazon Cognito + Twitter OAuth によるソーシャルログイン
- 一般ユーザーのアカウント登録

## Phase 3

### 4. 投稿申請・承認フロー

- 一般ユーザーがレギュレーションの投稿を申請できる
- 管理者が申請を承認・却下できる
- 承認後に公開される
