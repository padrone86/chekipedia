# データ設計

## 状態

設計中。詳細は各 Issue を参照。

## データの階層構造（未確定）

レギュレーションは以下の階層に紐づくと想定される。

```
グループ
  ├── メンバー（0人以上）
  └── イベント
        └── レギュレーション（1件以上）
```

未確定の問い：
- レギュレーションはグループ全体に紐づくか、メンバー個別か、またはその両方か
- メンバー情報は必須か（グループ単位でしか管理しないケースもある）
- イベントとレギュレーションは1対1か、1対多か

→ [Issue #1](https://github.com/padrone86/chekipedia/issues/1) 参照

## アクセスパターン（想定）

Phase 1 で必要なパターン：

1. グループ名（部分一致）で検索 → グループ一覧を返す
2. グループIDでレギュレーション一覧を取得
3. レギュレーションをCRUD（管理者のみ）

→ [Issue #2](https://github.com/padrone86/chekipedia/issues/2) 参照
