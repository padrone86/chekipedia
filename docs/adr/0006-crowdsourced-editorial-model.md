# ADR-0006: コンテンツ投稿に Wikipedia 的市民参加モデルを採用

- Date: 2026-06-13
- Status: Accepted

## Context

レギュレーション情報をどのように収集・管理するかの方針を決める必要があった。

## Decision

ファンによる自発的な投稿（Wikipedia 的市民参加モデル）を採用する。グループ運営も一般ユーザーの一人として扱い、特別扱いはしない。

## 却下した選択肢

### 運営による公式データ提供モデル
運営が積極的にデータを提供してくれる保証がない。また運営に特別なロールを与えると初期設計が複雑になる。

## Consequences

**メリット**
- 管理者（開発者）が全データを手入力する必要がなく、スケールしやすい
- 運営に特別なロールを与えないため、初期設計がシンプルになる

**デメリット・リスク**
- 投稿品質の担保は管理者の承認フローが必要（Phase 3 で対応）
- 投稿が集まらない可能性がある（コールドスタート問題）
