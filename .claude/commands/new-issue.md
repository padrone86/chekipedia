GitHub Issue を作成する。

手順：
1. 引数がある場合はそれをタイトルの候補として使う。なければユーザーに聞く
2. Issue の種別を確認する：
   - **バグ報告**：不具合・予期しない動作
   - **機能要望・設計タスク**：新機能・改善・設計の検討
3. 種別に応じたテンプレートで内容をユーザーと確認してから作成する

**バグ報告テンプレート**
- 何が起きたか
- 再現手順
- 期待する動作
- 環境（OS・ブラウザ等）

**機能要望・設計タスクテンプレート**
- 解決したい課題
- 提案する解決策（任意）
- 補足

4. `gh issue create --title "..." --body "..."` で作成する
5. 作成した Issue の URL をユーザーに伝える
